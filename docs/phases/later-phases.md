# Later phases

- **Phase 4 — report features (implemented)**: browser-local dashboard/history,
  Markdown/HTML/JSON exports, stateless share links, FAQ JSON-LD output, local
  settings, and result Open Graph metadata/image. See the two Phase 4 reports.
- **Phase 5 — optional account recovery (implemented under D-012)**:
  anonymous auditing remains available, while Supabase email-link auth can
  attach a device workspace to a verified account for cross-device history,
  reports, settings, provider tasks, and usage records. The older better-auth,
  Drizzle, and BYOK implementation remains archived at `backup/pre-rewrite`.
